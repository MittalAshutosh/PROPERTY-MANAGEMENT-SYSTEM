# Contributing to PMS

Thanks for your interest in improving PMS.

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install dependencies in both `pms-backend` and `pms-frontend`
4. Copy `.env.example` files and configure local values
5. Run backend and frontend locally

## Branching Convention

- `feat/<feature-name>`
- `fix/<bug-name>`
- `docs/<topic>`
- `chore/<task>`

## Commit Guidelines

Use clear, action-oriented commit messages.

Examples:

- `feat: add payment summary export`
- `fix: prevent lease form crash on empty data`
- `docs: improve backend setup section`

## Pull Request Checklist

- PR is focused and scoped
- Lint/build pass locally
- Changes are documented when needed
- Screenshots included for UI changes
- No secrets committed

## Code Style

- Keep components focused and composable
- Prefer clear names over short names
- Add comments only when logic is non-obvious
- Avoid unrelated formatting changes

## Reporting Issues

Open an issue with:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Environment details

We appreciate every contribution.
